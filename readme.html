1) Erstellen Sie ein Formular, in welchem ein Nutzer Daten eingeben kann. (5 P)

Das Formular soll folgende Elemente beinhalten. Die Aufgabe wurde in der quiz.html realisiert. Diese ist über die Navigation "Quiz" aufrufbar.

  • 2 Textfelder
    1. Erstes Textfeld : "Gib deinen Benutzernamen ein (mind. 5 Buchstaben)"
    2. Zweites Textfeld : "Gib deinen Vornamen ein (Pflichtfeld)"
  • Drop-Down-Menü mit mind. 6 Optionen
  • Wähle Schwierigkeitsstufe(max. 2 Angaben): 6 Schwierigkeitsstufen
  • Radiobuttons mit mind. 3 Optionen: Wähle eine Kategorie aus: mit Kategorien
  • 1 Feld zur Eingabe von Zahlen: Wähle bis zu 8 Fragen
  • Eine Datalist mit mind. 3 Einträgen: Wähle einen Bootstyp aus

2. Implementieren Sie Vorgaben für die Benutzereingaben (4 P)

Soweit möglich, verwenden Sie für die Bedingungen HTML-Attribute, andernfalls implementieren Sie Prüfungen mit JavaScript unter Aufgabe 3.

  • Textfeld 1 muss mindestens 5 Buchstaben lang sein: über Attribut: minlength="5", sowie über die Funktion check() geprüft, siehe bootsquiz.js
  • Textfeld 2 ist Pflichtfeld: wird mit dem Attribut required und in javascript (siehe bootsquiz.js) gesichert. Für diese Seite war es nicht sinnvoll einen Submit Button zu implementieren, da die Seite immer neu geladen wurde und die Eingabe, sowie die Ausgabe somit verworfen wurden. (als Alternative wurde Alert gewählt)
  • Es muss mindestens ein Radiobutton ODER/UND eine Dropdown-Option ausgewählt sein (Über die Funktion check() geprüft, siehe bootsquiz.js)
  • In der Drop-down-Liste dürfen nur max. 2 Einträge markiert werden (Über die Funktion check(), siehe bootsquiz.js)
  • Die eingegebene Zahl muss zwischen 1 und 8 liegen: über Attribut: min="1" und max="8" und über die Funktion check(), siehe bootsquiz.js

3. Nehmen Sie eine Formularvalidierung vor (20 P)

Markieren Sie die Textfelder mit gelbem Hintergrund bei fehlerhafter Eingabe. (formular.css, siehe bootsquiz.js)

Erstellen Sie eigene Fehlermeldungen für fehlerhafte Eingaben bei allen geeigeneten Formularfeldern. (siehe bootsquiz.js und kontakt_javascript.js auf der kontakt.html seite)

Prüfen Sie per JavaScript, ob die unter b) genannten Bedingungen erfüllt wurden, bevor das Formular abgesendet wird. Benutzen Sie für die Aufgabe checkValidity(), setCustomValidity() und das validityState-Objekt.

4. Geben Sie die Formulareingaben aus (15 P)

wurde in der bootsquiz.js mit den Funktionen 'inObjektUmwandeln' und 'inHTMLwiedergeben' umgesetzt

  • Speichern Sie die Formulardaten beim Absenden in ein JavaScript-Objekt
  • Serialisieren Sie das Objekt in JSON.
  • Geben Sie die Formulardaten auf der HTML-Seite aus, indem Sie das JSON-Objekt in ein JavaScript-Objekt zurückwandeln und die Key-Value-Paare unterhalb des Formulars ausgeben. Nehmen Sie entsprechende Änderungen am DOM vor.
  • Geben Sie auch zur Überprüfung den Inhalt des JSON-Objekts aus

5. Erweitern Sie Ihr Semesterprojekt (unbepunktet, muss aber bestanden werden). Nehmen Sie weitere Implementierungen für Ihr Semesterprojekt vor:

  • Das Layout Ihrer Anwendung sollte (soweit noch nicht geschehen) im groben und ganzen fertig sein (Feinschliff kann auch im Nachgang erfolgen)

    3. 3. Funktionalität: Preisrechner : Daten zu verschiedenen Booten werden aus einer JSON Datei ausgelesen. Der Zugriff auf die JSON-Datei

    6. 6. Funktionalität: Preisrechner : Der Preis für die gewählte Bootsklasse wird anhand der weiteren Eingaben 'Saison', 'Rabatt' und 'Mietdauer' berechnet und ausgegeben.
      zu finden in preise.js - function berechnePreis() - Zeile
    7. 7. Funktionalität: Kontakt : Kontaktformular erstellen und als Txt-Datei als mail versenden
